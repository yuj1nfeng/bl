## 世界观要素

### 基础设定

**时空背景：**
- 时代（古代/现代/未来）
- 地点（现实/虚构/异世界）
- 时间线（单一/平行/循环）

**社会结构：**
- 政治体制
- 阶级划分
- 权力分布
- 法律制度

**文化背景：**
- 宗教信仰
- 风俗习惯
- 价值观念
- 语言特点

### 特殊设定

**奇幻世界：**
- 魔法体系
- 种族设定
- 神兽传说
- 神器宝物

**科幻世界：**
- 科技水平
- 外星文明
- 人工智能
- 时空技术

**异能世界：**
- 能力来源
- 能力分类
- 使用限制
- 觉醒条件

## 构建原则

### 一致性原则

**内部逻辑：**
- 规则自洽
- 前后一致
- 没有矛盾

**示例：**
```
设定：魔法需要付出代价
贯穿：每次使用都要体现代价
不能：有时需要代价，有时不需要
```

### 可信度原则

**细节支撑：**
- 具体而非抽象
- 有历史渊源
- 有现实依据

**渐进揭示：**
- 不要一次性倾倒
- 通过故事展现
- 让观众慢慢了解

### 独特性原则

**差异化：**
- 找到独特切入点
- 避免老套设定
- 加入创新元素

**记忆点：**
- 标志性元素
- 独特规则
- 特色文化

### 规则清单

**必须明确的规则：**
1. 力量体系规则
2. 社会运行规则
3. 禁忌与限制
4. 特殊物品规则

## 设定展现技巧

### 展示而非告知

**错误示范：**
```
"这个世界有魔法，分为五个等级..."
（大段说明）
```

**正确示范：**
```
通过角色施法过程展现
通过战斗展现等级差异
通过对话透露规则
```

### 冰山理论

**露出水面（10%）：**
- 故事中直接展现的
- 观众能看到的

**水面之下（90%）：**
- 完整的设定文档
- 背景历史
- 未展现的规则

**作用：**
- 让创作者心中有数
- 保证一致性
- 为续作留空间

## 常见问题

### 问题 1：设定过于复杂

**症状：** 规则太多，观众记不住

**解决：**
- 简化核心规则
- 逐步揭示
- 用故事带动设定

### 问题 2：设定与故事脱节

**症状：** 设定很精彩，但故事不好看

**解决：**
- 设定服务故事
- 通过冲突展现
- 不要为设定而设定

### 问题 3：设定前后矛盾

**症状：** 前面说的和后面不一致

**解决：**
- 建立设定文档
- 写作前查阅
- 修改时注意一致性

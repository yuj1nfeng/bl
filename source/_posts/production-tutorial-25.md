## 输出前检查

### 画面检查

**检查清单：**
```
□ 无黑场闪烁
□ 无错误帧
□ 无穿帮镜头
□ 字幕正确
□ 无水印logo
□ 色彩统一
□ 无抖动问题
```

### 音频检查

**检查清单：**
```
□ 音量平衡
□ 无爆音杂音
□ 音画同步
□ 无静音段
□ 淡入淡出
□ 格式正确
```

### 技术检查

**检查清单：**
```
□ 分辨率正确
□ 帧率正确
□ 码率合适
□ 格式符合平台
□ 文件大小合适
□ 命名规范
```

## 输出设置

### 视频参数

**B 站参数：**
```
分辨率：1920x1080 或 3840x2160
帧率：24/25/30/60 fps
码率：6000-8000 kbps（1080p）
格式：MP4（H.264）
```

**抖音参数：**
```
分辨率：1080x1920（竖屏）
帧率：24/30 fps
码率：3000-6000 kbps
格式：MP4
```

### 音频参数

**通用参数：**
```
格式：AAC
码率：320 kbps
采样率：48kHz
声道：立体声
```

### 编码设置

**H.264 设置：**
```
Profile: High
Level: 5.1
Keyframe: 2 秒间隔
编码次数：2 次（VBR）
```

**H.265 设置：**
```
Profile: Main10
Level: 5.1
编码次数：2 次
兼容性更好
```

## 质量检查流程

### 初审

**检查内容：**
- 内容完整性
- 技术规范性
- 无明显错误

**检查人员：**
- 制作人
- 导演

### 复审

**检查内容：**
- 细节质量
- 艺术效果
- 平台规范

**检查人员：**
- 品质控制
- 平台审核

### 终检

**检查内容：**
- 最终确认
- 输出文件
- 备份归档

**检查人员：**
- 项目负责人

## 多版本输出

### 平台版本

**各平台版本：**
- B 站版本（横屏高清）

**版本管理：**
- 统一命名

- 版本记录

### 备用版本

**备用版本类型：**
- 无字幕版本

- 预告版本

**用途：**
- 海外发行
- 播客发布
- 宣传推广

## 文件管理

### 命名规范

**命名规则：**
```
项目_集数_版本_日期。格式

示例:
MH_EP01_Final_20260330.mp4
MH_EP01_Preview_20260330.mp4
```

### 存储结构

**文件夹结构：**
```
/Project_Name
  /01_Source      # 原始素材
  /02_Project     # 工程文件
  /03_Exports     # 输出文件
    /Draft       # 草稿版本
    /Final       # 最终版本
  /04_Archive     # 归档文件
```

### 备份策略

**3-2-1 原则：**
- 3 份副本
- 2 种介质
- 1 份异地

**备份内容：**
- 工程文件
- 输出文件
- 素材文件

## 常见问题

### 问题 1：输出后色彩变化

**原因：** 色彩空间不匹配

**解决：**
- 统一色彩空间
- 检查伽马值
- 适当提高饱和度

### 问题 2：文件太大

**原因：** 码率太高

**解决：**
- 降低码率
- 使用 H.265
- 适当降低分辨率

### 问题 3：平台审核不通过

**原因：** 内容或技术不符合规范

**解决：**
- 了解平台规范
- 提前自查
- 及时修改
